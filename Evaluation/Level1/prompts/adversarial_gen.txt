You are a test data generator creating adversarial queries for a medical AI retrieval system.
The system maps natural language queries to biosignal parameter keys (param_keys) from VitalDB.

Your task is to generate {n} adversarial queries of type: {adversarial_type}

=== ADVERSARIAL TYPE DEFINITIONS ===

"ambiguous"
  - Take a normal clinical query and REMOVE all parameter-identifying hints.
  - The resulting query must be so vague that the system cannot determine which parameters are needed
    without asking for clarification.
  - Expected system behavior: "clarify" (ask the user what they mean)
  - Example transform: "What is the mean HR when MAP < 65?" → "How was the patient doing at that point?"

"impossible"
  - Generate a query that requests a physiological signal that does NOT exist in VitalDB's parameter table.
  - The signal must sound plausible in a surgical context but is genuinely absent from the database.
  - Do NOT use any of the known parameters listed below.
  - Expected system behavior: "not_found"
  - Signals known to be absent: {unavailable_signals}

"confusing"
  - Generate a query where the same physiological indicator is measured by MULTIPLE devices,
    but the query does NOT specify which device to use.
  - The system should detect the ambiguity and ask for clarification.
  - Expected system behavior: "clarify"
  - Use the following confusing parameter groups (same physiological indicator, multiple param_keys):
{confusing_groups}

=== CONTEXT FOR THIS BATCH ===

adversarial_type    : {adversarial_type}
source_queries      : {source_queries}
query_style         : {query_style}

=== RULES ===

- Write in English only.
- Match the declared query_style for vocabulary and tone:
  * "doctor" — clinical terminology and professional phrasing
  * "data_scientist" — statistical/analytical language
  * "layperson" — plain everyday language
- Each query must be clinically plausible (could come from a real user).
- For "ambiguous": do NOT include any param_key, abbreviation, or physiological term that unambiguously identifies a parameter.
- For "impossible": do NOT reference any param_key that exists in VitalDB.
- For "confusing": do NOT specify a device name or device prefix in the query.
- Each query must be linguistically distinct.

=== OUTPUT FORMAT ===

Return a JSON array of exactly {n} objects. No markdown, no explanations outside the JSON.

[
  {{
    "query": "<adversarial natural language query>",
    "adversarial_type": "{adversarial_type}",
    "expected_behavior": "<clarify|not_found>",
    "generation_notes": "<why this query is adversarial and what makes it challenging>"
  }},
  ...
]
